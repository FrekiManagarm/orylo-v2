# Story 5.1: Landing Page Feature Cards - Trust Score & Auto-Whitelist

## Status

Draft

## Story

**As a** potential customer visiting the landing page,
**I want** to see "Smart Customer Memory" and "Auto-Whitelist VIPs" features highlighted,
**so that** I understand Orylo's unique differentiation from competitors like Stripe Radar.

---

## Acceptance Criteria

1. Add "Smart Customer Memory" feature card to `components/landing/features.tsx`
2. Add "Auto-Whitelist VIPs" feature card to `components/landing/features.tsx`
3. Each card includes: icon, title, description, stat/badge, tags
4. Cards integrated into existing grid layout (maintain responsive design)
5. Animations and interactions consistent with existing feature cards
6. Icons are relevant and visually appealing (Brain, Crown, Shield, etc.)
7. Copy is clear, non-technical, and benefit-focused
8. Design follows existing Shadcn/ui patterns and dark theme

**Recommended Icons:**
- Smart Customer Memory: `Brain` (lucide-react) → represents intelligence, memory
- Auto-Whitelist VIPs: `Crown` (lucide-react) → represents VIP status, premium

**Copy Guidelines:**
- Focus on merchant benefits: "Réduisez les faux positifs", "Zéro friction pour vos VIPs"
- Avoid technical jargon: Use "mémoire intelligente" instead of "Trust Score persistant"
- Emphasize differentiation: "Orylo se souvient, contrairement aux solutions standards"
- Keep descriptions concise: 2-3 sentences max

**Existing Patterns to Follow:**
- Card component from Shadcn/ui
- Framer Motion animations: `initial`, `whileInView`, `viewport={{ once: true }}`
- Hover effects: gradient overlay, border color change
- Grid layout: responsive with Tailwind breakpoints
- Dark theme: zinc-900 backgrounds, white text, indigo/purple accents

**Critical Constraints:**
- No breaking changes to existing feature cards
- Responsive design must work on all screen sizes
- Animations must be performant (no jank)
- Icons must be clear and recognizable
- Copy must be in French

### Testing

**Test Framework:** Manual visual testing

**Testing Standards:**
- Visual testing on multiple browsers
- Responsive testing on different screen sizes
- Animation testing (scroll into view)
- Hover effect testing

**Test Coverage Requirements:**
- All new feature cards render correctly
- Grid layout adapts to screen size
- Animations trigger on scroll
- Hover effects work

**Test Scenarios:**
1. Desktop viewport → 3 columns, cards aligned, animations smooth
2. Tablet viewport → 2 columns, responsive layout
3. Mobile viewport → 1 column, readable text, no overflow
4. Scroll into view → animations trigger (fade in + slide up)
5. Hover over card → gradient overlay appears, border glows
6. Icons render correctly (Brain, Crown)
7. Tags display properly (wrapped on mobile)
